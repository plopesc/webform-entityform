Exposing forms to end users

  • Collect extra information
  • Get feedback from our users
  • Event registration
  • Application forms
  • Information not considered as "website content"

Drupal alternatives

  • Form API
  • Generic content type
  • Embed an external survey system form
  • External application sync modules
  • Survey form modules

Survey form modules

  • Allow to create survey forms to site admins
  • Store submissions results for further analysis
  • Support for notification to admins and users
  • Access control to the form

Webform vs. Entityform

Webform

  • Created in 2004
  • More than 450K installations
  • Almost 150 releases since Drupal 4.4.x
  • De facto standard survey forms module

https://www.drupal.org/project/webform

Webform key features

  • Survey forms system on top of Drupal
  • Easy to use for non-experienced users
  • Single DB schema & scalability
  • Built-in support for usual tasks
  • Hundreds of add-on modules

Entityform

  • Created in 2011
  • More than 15K installations
  • Ceated for D7 and based on its architecture
  • Fully integrated with Drupal core subsystems

https://www.drupal.org/project/entityform

Entityform key features

  • Use Drupal entities to provide survey forms
  • Requires experience managing Drupal sites
  • Scalability issues with 100+ fields
  • Requires other modules for some usual tasks
  • Easy integration with most used modules

Fight!

Storage & scalability

WebForm EntityForm
DB Storage Custom Table Drupal Field System
Max # items Hundreds < 100
Max # forms Hundreds < 50

Form Creation

WebForm EntityForm
Creation Attached to a node Own page
Submissions access Built-in Built-in
Confirmation page & redirect Built-in Built-in
Submissions limit Built-in Built-in, basic
Draft support Built-in Built-in
Available as block Built-in Contrib (Entityform Block)

Form Fields

WebForm EntityForm
Field types 9 Built-in + Contrib Any Field API field type
Conditional Fields Built-in Contrib (Conditional Fields)
Grouping Fields Built-in (Fieldset) Contrib (Field Group)
Multipage Built-in Contrib (Field Group)

Survey features

WebForm EntityForm
Analytics Built-in Contrib (Views)
Data export Built-in Contrib (Views Data Export)
Notifications Built-in Contrib (Rules)
Charts Contrib (Webform Charts + Charts) Contrib (Views + Charts)

Integration

WebForm EntityForm
Entity API Not supported Built-in
Views Built-in, custom Built-in
Rules Contrib (Webform Rules) Built-in
Features Contrib (UUID based) Built-in
Multilingual Contrib (Webform Localization) Contrib (Internationalization)

Conclussions

  • There is no a golden hammer solution
  • Webform is a survey form specific tool
  • Entityform adapts Entity API to survey forms

It's necessary to study project requirements before making your final decision

Helpful questions

  • How many forms will be necessary?
  • How many fields will need any form?
  • Will be the site multilingual?
  • Will be form admins familiar with Drupal administration?
  • What kind of field types will I need?
  • Will I need complex field grouping or multipage support?

Helpful questions

  • How survey submissions will be extracted/processed?
  • Do I need complex submission notification logic?
  • Will I need to deploy forms between environments?
  • How dynamic will be the forms?
  • Will be necessary a complex submission limit logic?
  • Where the forms will be placed?

Drupal 8

Back to the survey form future

Drupal 8 Webform

  • New team of maintainers
  • Need to rewrite and redesign
  • Specific survey form tool on top of drupal
  • Contrib ecosystem needs to be rewritten then

Drupal 8 EForm

  • Renamed to EForm to avoid naming issues
  • Ted bowman continue leading the project
  • Need to update the module to the D8 Entity architecture
  • Former required modules are in core (Entity API, Views)
  • Helpful modules are being ported (Rules, Field Group)
